## How to use this glossary

- New domain terms must be added here **in the same change** that introduces them
  (CLAUDE.md). PRD-001 wins on any conflict — fix this doc, not the PRD.
- Code identifiers follow the todos-for-dues naming style: snake_case plural tables
  (`users`, `user_role_transitions`), snake_case columns, PascalCase TypeScript types,
  camelCase helpers, SCREAMING_SNAKE env vars. Identifiers for not-yet-built tables are
  prescriptive: designs use them or amend this row in the same change.
- `R-NN` / `AC-NN` / `Q-NN` references are PRD-001 IDs; `BC-NN` references are DDD-002.

## Identity & access

| ID | Term | Definition | Code identifier | Notes |
|----|------|------------|-----------------|-------|
| T-01 | User | A person known to the app. A row is auto-created in the Default Role (T-48) on first successful Authentik login — no approval gate (R-03). | `users` | Better Auth schema + `role_id` FK (was a `role` enum column, superseded by ADR-012); displayName/email from OIDC claims (AC-02). |
| T-02 | Role | **Amended by ADR-012:** an admin-managed, DB-backed named group with an editable app set (T-46). A User has **exactly one** (`users.role_id`). No longer a `Member`/`Admin` text enum — that is gone. Two seeded System Roles (T-47): **Admin** (T-04, superuser) and **Default** (T-48, new-user role); admins create more. Role assignment writes Audit Rows (`user_role_transitions`) in the same transaction (R-04). | `roles`, `users.role_id` | The old `enums.ts` `ROLES` array and `users.role` CHECK are removed (migration 0007). Session carries `role = { id, name, isAdmin }`. |
| T-03 | ~~Member~~ | **Superseded by ADR-012:** there is no `Member` role. The role every new login lands in is now the **Default** system role (T-48); "basic user" = a user in the Default role. | ~~`'Member'`~~ | Retained for history; use Default (T-48) / Role (T-02). |
| T-04 | Admin | **Amended by ADR-012:** the **superuser Role** (`roles.is_admin`) — implicitly sees every App (no `role_app_grants` rows; new Apps included automatically), grants admin-console access, and is immutable (no rename/edit/delete). First claimed via Bootstrap Admin (T-06); the last Admin-role member can't be moved off it (LastAdminError). | `roles.is_admin` (session `role.isAdmin`) | PRD-001 Actors & roles; R-02. `adminProcedure` gates on `role.isAdmin`. |
| T-05 | Family | **Amended by ADR-012:** no longer a flag/designation — it is an example Role (T-02), now **seeded by migration 0007** as a normal (editable/deletable) role granting every App except Tautulli. Phase-3 Family-Only-Library gating (R-26/R-27) will attach to a **role attribute**, not the removed `users.is_family`. | seeded row in `roles` (name `Family`; `is_admin`/`is_default`/`grants_all` all false) | `users.is_family` / `tags.is_family` are dropped (migration 0007); see ADR-012 C-09/C-12 and T-21. |
| T-06 | Bootstrap Admin | Idempotent promotion to Admin of a first login whose email (case-insensitive) is on the allowlist; repeat logins are no-ops (R-02, AC-03). | `BOOTSTRAP_ADMIN_EMAILS` (env) | Allowlist seeded with both owner emails (Q-01). |
| T-07 | Session | The authenticated browser session established after the Authentik OIDC round-trip; 7-day cookie (AC-01). | `sessions` (Better Auth) | No password form exists anywhere (R-01). |
| T-08 | Authentik | The OIDC identity provider at `authentik.haynesnetwork.com` — the only sign-in path. Plex is offered as the credential inside Authentik (R-01). | `AUTHENTIK_*` (env) | Syncing permissions into Authentik is a follow-on (R-30). |
| T-46 | Role App Grant | (ADR-012) A `(role, app)` pair — the editable app set a Role grants (T-13). The Admin superuser role holds **no** rows here (implicit all-apps). Composite PK dedupes; both FKs cascade. | `role_app_grants` | Replaces `user_app_grants` + `tag_app_grants` + `app_catalog.default_visible`. |
| T-47 | System Role | (ADR-012) One of the two seeded, fixed-id, non-deletable Roles: **Admin** (T-04, `is_admin`) and **Default** (T-48, `is_default`). Partial unique indexes enforce at most one of each; a CHECK forbids a role being both. Migration 0007 seeds a **third** role, `Family` (T-05), but it is a *normal* role, not a System Role — editable/deletable. | `roles.is_admin` / `roles.is_default` (`SEEDED_ROLE_IDS`) | Fixed ids so `users.role_id` can DB-default to Default. Domain writers reject renaming/deleting them (SystemRoleImmutableError). |
| T-48 | Default (Role) | (ADR-012) The System Role assigned to every new User (`users.role_id` DB default). Its seeded app set is **Seerr, Plex, K8Plex, and PlexOps** (T-10, migration 0007); the app set and description are editable but the role can't be renamed or deleted. | `roles.is_default` | The successor to the old "Member" default (T-03). |
| T-49 | All-Apps Role | (ADR-012, shipped after) A non-Admin Role with `roles.grants_all` = true: grants **every** catalog App — including ones added later — and holds **no** Role App Grants (T-46), i.e. Admin's (T-04) all-apps reach **without** admin-console access. Effective Permissions (T-16) = all apps when `is_admin OR grants_all`. Surfaced as the **"All apps"** checkbox on `/admin/roles` (greys out + disables the per-app list). | `roles.grants_all` | `createRole`/`updateRole` take `grantsAll` (true clears the explicit app set); `RoleInput`/`RolePatchInput` carry it; `roles.list` returns it. |

## Plex servers & libraries

| ID | Term | Definition | Code identifier | Notes |
|----|------|------------|-----------------|-------|
| T-17 | Plex Server | One of the three Plex instances under management, keyed by owner slug: `haynestower` (legacy Unraid), `haynesops` and `hayneskube` (k8s) (R-25; OPS-002). | T-18 | Plex Library | A library on one specific Plex Server; **identity is `(server_id, section_key)`, never name** (Q-03 resolved: servers mirror Movies/TV under different names). | `plex_libraries` (built; UNIQUE(`server_id`,`section_key`)) | Soft-state `available` flag — a vanished library is withheld, not deleted (DESIGN-007 D-04). |
| T-19 | Library Grant | **Redefined by ADR-017:** a **Role → Library** grant — permission for members of a Role to self-add a given Plex Library (R-25..R-27). Mirrors T-46 Role App Grant. | `role_library_grants` (built) | Share application happens in Plex Sharing (BC-04); grant edits audited (`update_role_libraries`). |
| T-20 | Allowed Library Set | A User's effective set of self-addable libraries. **Amended by ADR-024:** for a non-admin user it is the UNION of their Role's explicit `role_library_grants` and every available library on any server their Role holds a Server All-Grant (T-67) on; Admin ⇒ all available libraries. `grants_all` does NOT imply all libraries (ADR-017 C-03). | `EffectiveLibrary` + `effectiveAllowedLibrariesForUser()`; `allGrantedServerIdsForUser()` (the all-toggle gate) | Re-derived inside each share mutation (TOCTOU gate). |
| T-21 | Family-Only Library | **Redefined by ADR-017:** an ordinary Plex Library granted only to the `Family` Role — there is **no** `plex_libraries.is_family_only` column. Exactly `HNet Home Videos` and `HNet Photos` (HAYNESTOWER) today (R-26; C-09/C-12). | rows in `plex_libraries` referenced only by `role_library_grants` for the Family role | Family self-service is just its role's grant set. |
| T-52 | Plex Share | An applied library share on a specific Plex Server — the user's `SharedServer` on the server owner's plex.tv account. Usually a **per-section** grant (a set of shared sections) mutated via read-merge-write (never blind overwrite). **Amended by ADR-024:** a `SharedServer` can also be **all-libraries** (`allLibraries="1"` — share-everything, incl. future libraries); a user whose Role holds a Server All-Grant (T-67) self-toggles between that state and an explicit list. Not every share is per-section — see T-67. |
| T-53 | Library Registry Refresh | The admin-triggered sync that reads each server's `GET /library/sections` (+ `/identity`) and upserts `plex_servers`/`plex_libraries` keyed on `(server_id, section_key)`, marking unseen libraries `available=false` (never deleting). **Degrades per server** (DESIGN-007 D-12): each server refreshes in its own transaction and a Plex failure on one is folded into a per-server summary (`{ ok, servers: [{ slug, name, ok, libraryCount?, error? }] }`) — it never aborts the others. | `refreshPlexRegistry` / `plex.refreshRegistry` | Admin-button only for now — a CronJob is future work (Q-06). | T-54 | Machine Identifier | A Plex Server's stable GUID; the key the plex.tv sharing API addresses a server by (`/api/servers/{machineIdentifier}/…`). Distinct from the owner slug and the ingress subdomain. | `plex_servers.machine_identifier` | Verified live 2026-07-06; pinned in `PLEX_MACHINE_IDENTIFIERS`, re-read by the registry refresh. |
| T-67 | Server All-Grant | (ADR-024) A `(role, server)` grant meaning the Role grants ALL libraries — including future ones — on that Plex Server. A member of such a role self-toggles their own account between the plex.tv **all-libraries** state and an **explicit list** (leaving All seeds the current full set — no access loss; `setServerAllShare`). An `is_admin` role holds no rows and implies all-on-every-server. | `role_plex_server_all_grants` (migration 0015); `allGrantedServerIdsForUser`; `setServerAllShare` | Written by `setRoleLibraries` (replace-whole-set, audited `update_role_libraries`). Server-scoped share events `share_all_enabled`/`share_all_disabled` (T-52). |

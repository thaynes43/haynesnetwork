Interim source of truth until PRD-001 and ADR-001..010 land; after that, those docs win.

## Product intent (owner's words, condensed)

SSO navigation hub for `*.haynesnetwork.com` apps plus advanced media features that
user-facing apps like Seerr don't provide: look up media, "fix" broken content by forcing
the *arr to re-pull, a ledger of who requested what / what was deleted when, capture of
wanted-but-not-on-disk, and a critical-failure failsafe where ledger lists can be synced
back into Sonarr/Radarr/Lidarr to re-add lost content or bootstrap a fresh server. The
*arrs remain the source of truth; the app DB is synced from them.

## Permission model (from conversation)

- Roles: first-login-bootstrap Admin; everyone else defaults to standard user.
- Admins grant per-user app visibility (which tiles render).
- Plex libraries: users self-add/remove libraries on their account within what an admin
  allows; default = all libraries EXCEPT `HNet Home Videos` and `HNet Photos`, which only
  users designated **family** can see/add.
- Admin-created **tags** bundle a common set of permissions and are applied to users to
  save per-user setup; per-user (un-tagged) permission setting still works.
- Authentik itself has no access-control enforcement for these yet; syncing permissions to
  Authentik's API is a follow-on. Exposing/hiding links is the accepted starting point.
